# Security Policy

## Supported versions

Security reports should target the latest released version of PHOTO-CAT.

## Reporting a vulnerability

Please do not publish security-sensitive issues publicly before maintainers have had a reasonable chance to review them.

Report suspected vulnerabilities privately to the repository owner or through the private contact channel listed by the project.

Include:

- affected PHOTO-CAT version
- operating system
- steps to reproduce
- expected and actual behaviour
- whether the issue exposes data, executes code unexpectedly, or affects generated outputs

## Scope

PHOTO-CAT is a local scientific analysis tool. Reports are most useful when they concern local file handling, dependency setup, unsafe execution behaviour, or handling of untrusted input files.
